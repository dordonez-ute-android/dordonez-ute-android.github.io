Soporte de geopackage para osmdroid


¿Qué es Geopackage?


Básicamente, es una base de datos sqlite que puede contener:
            • 0 o más fuentes de mosaicos ráster (piense en jpg o png)
            • 0 o más fuentes de características (piense en gráficos vectoriales)

Geopackage utiliza internamente un sistema de numeración de mosaicos diferente al de osmdroid o cualquier otra cosa basada en el formato de mosaico deslizante (z / x / y), por lo que se requiere una biblioteca de soporte para cortar, reorganizar y transformar lo que esté en la base de datos del geopaquete en algo más utilizable.

Administrador de caché

Originalmente escrito por el usuario MKer como parte de osmbonuspack (migrado en algún lugar alrededor de la versión 5.3), Cache Manager proporciona una serie de funciones geniales. Aquí hay algunos aspectos destacados rápidos:

  •     • Descargar y almacenar en caché mosaicos para un área geográfica determinada (límites y zoom mínimo / máximo)
  •     • Descargar y almacenar en caché mosaicos para un conjunto determinado de puntos de paso (conjunto de puntos y zoom mínimo / máximo)
  •     • Descargue y produzca un archivo de mosaicos persistente (función similar a MOBAC) (a través de límites o puntos de referencia
  •     • Limpiar la caché de un área o puntos de paso determinados.

OSMDroid CacheManager mejorado para descargar mosaicos de mapas cubiertos por una lista de waypoints. OSMDroid CacheManager disponible desde 5.3, le permite descargar mosaicos de mapas basados en un cuadro delimitador. Por supuesto, eso es bastante útil, ya que sabemos que hay muchos lugares donde la conexión móvil no es lo suficientemente buena para usar el mapa en línea.




 


WMS


Un sistema de gestión de almacenes (WMS) es una aplicación de software que da soporte a las operaciones diarias de un almacén.


Características

  •     •Permiten la gestión centralizada de tareas, como el seguimiento de los niveles de inventario y la ubicación de existencias.
  •     •Pueden ser aplicaciones independientes o pueden estar integrados en un sistema de Planificación de Recursos Empresariales (ERP).

Capas de WMS


Una capa de servicio WMS está formada por tres entradas que se organizan jerárquicamente en el panel Contenido:

  • • En la parte superior se encuentra el nombre del servicio WMS, quecontiene todas las capas del mapa WMS.
  • • El siguiente nivel es el de las capas del grupo WMS, que tienen como única función organizar las subcapas WMS en grupos relacionados. Hay al menos una capa de grupo WMS, pero puede haber cualquier cantidad de capas de grupo WMS (e incluso grupos que se encuentren dentro de otros grupos). Los grupos de capas WMS no contienen capas de mapas.
  • • El tercer grupo, subcapas WMS, es el que realmente contiene las capas de mapas.



Requisitos






  • • El servidor WMS debe admitir la producción de un mosaico para un cuadro delimitador específico con un tamaño de píxel específico.
  • • No se admiten capas que requieren parámetros de entrada adicionales, como desplazamientos de tiempo.
  • • No se admiten las capas que requieren un mensaje de publicación HTTP.
  • • No se admiten las capas que utilizan píxeles no cuadrados (es decir, la altura y el ancho de la imagen no son iguales).
  • • No se admiten las capas que requieren un mensaje de publicación HTTP.

El adaptador de MapsForge implementa el patrón de proveedor de mosaicos.

Osmdroid utiliza dos componentes para mostrar imágenes de mapas: el proveedor de mosaicos y el origen de mosaicos.
El proveedor de mosaicos se utiliza para determinar cómo cargar mosaicos (en línea, fuera de línea, carpetas de activos, etc.)

¿Qué es Ground Overlay?


  • • La clase Ground Overlay le permite trazar una imagen en el mapa si conoce las coordenadas latitud y longitud de las cuatro esquinas de la imagen.

  • • Esta clase fue añadida en la versión 6.0.0 de osmdroid para casos de uso como Imágenes de satélite, meteorología / radar, etc.

Arquitectura de proveedor de mosaicos modular


  • • Osmdroid utiliza dos componentes para mostrar imágenes de mapas, el proveedor de mosaicos y el origen de mosaicos.

  • • El proveedor de mosaicos se utiliza para determinar cómo cargar mosaicos

  • • La fuente de mosaicos determina qué conjunto de imágenes se muestra.



Proveedor de mosaicos de nivel superior


La arquitectura del proveedor de mosaicos comienza con un proveedor de mosaicos de nivel superior que administra las llamadas para obtener mosaicos. Hay dos proveedores de teselas de nivel superior para los usuarios de SDK:


MapTileProviderBasic


Este proveedor de mosaicos de nivel superior implementa una cadena de solicitud de mosaicos predeterminada que incluye un MapTileFilesystemProvider (un caché del sistema de archivos), un MapTileFileArchiveProvider (proporciona mosaicos de archivos) y un MapTileDownloader (descarga mosaicos de mapas desde el servidor HTTP).







MapTileProviderArray (extiende MapTileProviderBasic)


Este proveedor de mosaicos de nivel superior permite que un usuario de SDK proporcione una serie de proveedores de mosaicos asincrónicos modulares que se utilizarán para obtener mosaicos de mapas. Cuando se solicita un mosaico, MapTileProviderArray primero verifica MapTileCache (de forma sincrónica) y devuelve el mosaico si está disponible.



Módulos de proveedor de mosaicos


MapTileProviderArray es la base de la arquitectura modular de mosaicos y es la clase de "administrador" que maneja las llamadas a los módulos del proveedor de mosaicos. Los módulos de proveedor de mosaicos proporcionados disponibles para su uso son:

  • •MapTileFilesystemProvider (extiende MapTileModuleProviderBase): Implementa un caché del sistema de archivos y proporciona teselas en caché. Esto funciona como un proveedor de mosaicos al proporcionar mosaicos en caché.

  • •MapTileFileArchiveProvider (extiende MapTileFileStorageProviderBase): Este proveedor de mosaicos puede proporcionar mosaicos almacenados en un archivo de archivo.

  • •MapTileDownloader (extiende MapTileModuleProviderBase): Carga mosaicos desde un servidor HTTP.

  • •MapTileApproximater (extiende MapTileModuleProviderBase): Este módulo intentará llenar los huecos de cobertura de mosaicos utilizando imágenes de mosaicos disponibles con el zoom actual-1 al zoom mínimo.



Fuentes de Mosaicos


Los proveedores de mosaicos utilizan clases de fuentes de mosaicos para encapsular información sobre el tipo de mosaicos que atiende el proveedor. Una fuente de mosaico también proporciona servicios de renderizado para crear elementos de diseño.

Las fuentes de mosaicos que proporcionan imágenes rasterizados (mapa de bits, jpg, png, gif, etc.) pueden extender la clase BitmapTileSourceBase.



Fuentes de mapas en líneas


Osmdroid viene con algunas fuentes de mapas preconfiguradas para su uso. Las reglas de acceso al uso varían según la fuente



Mosaicos de mapas sin conexión


Osmdroid proporciona soporte listo para usar para varios tipos diferentes de cachés de mosaicos de mapas fuera de línea.
