Descripción general de los servicios

Un Service es un componente de una aplicación que puede realizar operaciones de larga ejecución en segundo plano.

  • •No proporciona una interfaz de usuario.

  • •Puede iniciar un servicio y continuar ejecutándose en segundo plano aunque el usuario cambie a otra aplicación.

  • •Un componente puede enlazarse con un servicio para interactuar con él e incluso realizar una comunicación entre procesos (IPC).


Estos son los tres tipos diferentes de servicios:




Primer Plano


  • • Un servicio en primer plano realiza una operación que el usuario puede notar.

  • • Los servicios en primer plano deben mostrar una notificación.

  • • Estos servicios continúan ejecutándose incluso si el usuario deja de interactuar con la aplicación.












Segundo Plano


• Realiza una operación que el usuario no nota directamente.




Enlace


  • • Un servicio es de enlace cuando un componente de la aplicación se vincula a él llamando a bindService().

  • • Ofrece una interfaz cliente-servidor que permite que los componentes interactúen con el servicio, incluso comunicación entre procesos (IPC).

  • • Se ejecuta solamente mientras otro componente de aplicación está enlazado a él.

  • • Se pueden enlazar varios componentes con el servicio a la vez, pero el servicio se destruye cuando todos ellos se desenlazan.









Diferencia entre un servicio y subproceso


Servicio


Un servicio es simplemente un componente que puede ejecutarse en segundo plano, incluso cuando el usuario no está interactuando con tu aplicación.


Declarar un servicio en el manifiesto


Se debe declarar todos los servicios en el archivo de manifiesto de la aplicación, tal como se hace con las actividades y otros componentes. Para declarar tu servicio, agrega un elemento como campo secundario del elemento de .





Crear un servicio iniciado


  • • Un servicio inicia llamando a startService(), lo cual da como resultado una llamada al método onStartCommand() del servicio.

  • • Cuando se inicia un servicio, tiene un ciclo de vida independiente del componente que lo inició.

  • • Se detienen a sí mismo llamando a stopSelf(), u otro componente puede detenerlo llamando a stopService().

  • • Un componente de una aplicación, como una actividad, puede iniciar el servicio llamando a startService() y pasando una Intent que especifique el servicio e incluya datos para el servicio.

  • • El servicio recibe esta Intent en el método onStartCommand().


Iniciar un servicio


Puedes iniciar un servicio desde una actividad u otro componente de la aplicación pasando una Intent a startService() o startForegroundService().

El sistema de Android llama al método del servicio onStartCommand() y le pasa la Intent, que especifica qué servicio iniciar.



Crear un servicio enlazado


Un servicio enlazado es un servicio que permite que los componentes de la aplicación se enlacen a él llamando a bindService() para crear una conexión a largo plazo.

Para crear un servicio enlazado:

  • • Se debe implementar el método de devolución de llamada onBind().

  • • Otros componentes de la aplicación pueden llamar a bindService() para recuperar la interfaz y comenzar a llamar a métodos en el servicio.

  • • El servicio está activo solo para servir al componente de la aplicación que está enlazado a él.

  • • Se debe definir la interfaz que especifica cómo un cliente puede comunicarse con el servicio. Esta interfaz entre el servicio y un cliente debe ser una implementación de IBinder .

  • • Cuando un cliente termina de interactuar con el servicio, llama a unbindService() para desenlazarse. Cuando ya no hay clientes enlazados con el servicio, el sistema lo destruye.

Enviar notificaciones al usuario.


Cuando se está ejecutando, un servicio puede notificar al usuario acerca de los eventos utilizando notificaciones del sistema o notificaciones de la barra de estado.





Ejecutar un servicio en primer plano


Un servicio en primer plano es un servicio del que el usuario está activamente al tanto y que no es candidato a que el sistema lo cierre cuando tiene poca memoria.

  • • Debe proporcionar una notificación en la barra de estado, que aparece debajo del encabezado En curso.

  • • Para solicitar que tu servicio se ejecute en primer plano, llama a startForeground(). Este método tiene dos parámetros: un valor entero que identifica de forma exclusiva la notificación de barra de estado. La prioridad de la notificación debe ser PRIORITY_LOW o superior.



Administrar el ciclo de vida de un servicio


El ciclo de vida del servicio puede seguir dos rutas de acceso diferentes:


Servicio Iniciado


  • • El servicio se crea cuando otro componente llama a startService().

  • • El servicio se ejecuta de forma indefinida y se debe detener a sí mismo llamando a stopSelf().

Servicio enlazado


  • • El servicio se crea cuando otro componente (un cliente) llama a bindService().

  • • Luego, el cliente se comunica con el servicio a través de una interfaz IBinder.

  • • El cliente puede cerrar la conexión llamando a unbindService().

  • • Se pueden enlazar varios clientes al mismo servicio y, cuando todos ellos se desenlazan, el sistema destruye el servicio.


Servicios de primer plano


Definición

  • • Los servicios de primer plano realizan operaciones que son perceptibles para el usuario.

  • • Cada servicio en primer plano debe mostrar una notificación en la barra de estado que tenga una prioridad igual PRIORITY_LOW o superior. La notificación no se puede descartar a menos que el servicio se detenga o se elimine del primer plano.


Solicitar el permiso de servicio en primer plano


Los servicios en primer plano deben solicitar el FOREGROUND_SERVICE permiso, como se muestra en el siguiente fragmento de código.


Iniciar un servicio de primer plano


Para solicitar que su servicio se ejecute en primer plano, llame a startForeground(). Este método toma dos parámetros: un número entero positivo que identifica de forma única la notificación en la barra de estado y el Notification objeto en sí.



Declarar tipos de servicios en primer plano


Para quitar el servicio del primer plano, llame stopForeground(). Este método toma un booleano, que indica si se debe eliminar también la notificación de la barra de estado. Tenga en cuenta que el servicio continúa ejecutándose.


Eliminar un servicio del primer plano


Si se accede a información de ubicación en un servicio en primer plano, declare el location tipo de servicio en primer plano como un atributo de su componente.
Si se accede a la cámara o al micrófono en un servicio en primer plano, declare los tipos de servicio camera o en microphone primer plano, respectivamente, como atributos de su componente.
