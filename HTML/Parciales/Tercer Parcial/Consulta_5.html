Webview

Contenido basado en la Web


Para presentar contenido propio y de confianza a un usuario, una buena opción es hacer uso de WebView el cual permite que el contenido web esté disponible para los usuarios de dos maneras: en un navegador web tradicional y en la aplicación móvil.




Permite especificar las propiedades de viewport y estilo que hacen que las páginas web aparezcan con el tamaño y la escala adecuados en todas las configuraciones de pantalla de los principales navegadores web.




Incluso se puede definir una interfaz entre la app para Android y la página web que permita que JavaScript en las páginas web llame a las API en la app si se proporciona API de Android a la aplicación basada en la Web.

Sin embargo, no se debe desarrollar una app de Android como un medio para visualizar un sitio web. En lugar de eso, las páginas web incorporadas en la app deben diseñarse especialmente para ese entorno. Debe tomarse en cuenta la compatibilidad de pantallas, para ello se toma en cuenta dos factores:

• ViewPort: Área rectangular que proporciona una región para elementos de diseño a la página web.

• Densidad de pantalla: Tanto los navegadores como WebView convierten valores de píxeles CSS a valores de píxeles independientes de la densidad por lo que las páginas se perciben del mismo tamaño en cualquier pantalla. Android Studio maneja tres densidades: pequeña, mediana y grande

 


Alternativas a WebView


Existen alternativas que pueden ofrecer funciones similares con ventajas, tales como; requerir menos configuración, carga y funcionalidades más rápidas, brindan protecciones de la privacidad mejoradas y acceden a las cookies del navegador. Estas alternativas pueden ser usadas si se quiere:

• Dirigir a los usuarios a un sitio móvil que compila una aplicación web progresiva.

• Mostrar contenido web de terceros enviando un intent a los navegadores web instalados.

• Evitar salir de la aplicación o personalizar la interfaz de usuario del navegador usando las pestañas personalizadas de Chrome

 









 


Algunos ejemplos de alternativas a WebView son:



• Crosswalk view:

Se trata de una aplicación en tiempo de ejecución basada en Chromium/Blink. Es Open Source y fue desarrollado por el Centro de Tecnología de Código Abierto de Intel, aun que actualmente ya no le dan mantenimiento.

• Chrome Custom Tabs:

Las pestañas personalizadas de Chrome brindan a las aplicaciones más control sobre la experiencia web y hacen que las transiciones entre contenido nativo y web sean más fluidas sin tener que recurrir a WebView. Permiten al desarrollador preiniciar Chrome y precargar contenido para más rapidez al mostrar el contenido.



 


WebView



La clase WebView es una extensión de la clase View que permite mostrar contenido web como parte del diseño de una actividad. No incluye las funcionalidades del navegador web como controles de navegación o barra de direcciones, únicamente muestra una página web. Está pensado para consultas puntuales desde la app, esto ayuda a que la carga de páginas sea más rápida.





Situaciones en las que conviene usar WebView:

• Cuando se desea proporcionar información actualizable, como un acuerdo de usuario final o una guía de usuario.

• Si la app proporciona al usuario datos que siempre requieren una conexión a Internet para recuperar información, como el correo electrónico.

Administrar objetos WebView

Android proporciona API’s para administrar objetos WebView. Al agragarlas de manera correcta, mejoran la estabilidad y seguridad de la app. Desde Android 7 (API 24) existen diversos paquetes como la biblioteca WebKit que incluye el método getCurrentWebViewPackage(), el mismo que permite obtener información relacionada con el paquete que muestra contenido web en la app, a la vez que maneja errores.





Para que los usuarios obtengan una experiencia de navegación más segura, los WebView verifican las URL mediante la Navegación segura de Google. Desde Android 8.0 (API 26), se usa el método setSafeBrowsingEnabled() para activar o desactivar la Navegación segura en un objeto WebView individual.
