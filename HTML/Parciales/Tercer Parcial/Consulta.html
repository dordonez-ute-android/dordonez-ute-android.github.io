UBICACIÓN, GPS

Introducción


El reconocimiento de la ubicación le permite a una aplicación brindar una experiencia más contextual, debido a que los usuarios llevan sus dispositivos móviles a todas partes. Las API de ubicación disponibles en los Servicios de Google Play permiten agregar la función de reconocimiento de la ubicación con seguimiento automático de la ubicación, geovallado y reconocimiento de actividades.

Configurar una aplicación con servicios de ubicación


Permisos


El primer paso es solicitar permisos de ubicación en el Android Manifest. El sistema de Android incluye varios permisos relacionados a la ubicación que deben ser solicitados dependiendo de los requisitos de la aplicación.

Las dos categorías existentes son:

• Ubicación en primer plano: Si la aplicación contiene una función que comparte o recibe información de ubicación solo una vez o durante un tiempo definido.

Es necesario declarar un servicio en primer plano primero.




A continuación, se declaran los permisos dependiendo de la precisión requerida. ACCESS_COARSE_LOCATION; proporciona una precisión de ubicación de hasta una manzana. ACCESS_FINE_LOCATION; proporciona una ubicación más precisa y es necesario para algunas tareas de conectividad.




Ubicación en segundo plano:

Si una función dentro de la aplicación comparte constantemente la ubicación con otros usuarios o usa la API de geovallado.

Se debe declarar el permiso ACCESS_BACKGROUND_LOCATION en el Android Manifest.



 


Para solicitar estos permisos al usuario, se debe esperar a que interactúe con la aplicación y de preferencia debe realizarse de manera incremental, de manera el usuario tenga más control y transparencia al usar la aplicación.

 




En el caso de los permisos en segundo plano, se debe tener distintas consideraciones para versiones superiores a Android 10. Hasta esta versión, el usuario puede activar la función de ubicación en segundo plano de manera más sencilla desde la misma aplicación, sin embargo, desde Android 11 en adelante el usuario debe habilitar el servicio en configuraciones.

Mediante el método getLastLocation() se solicita la última ubicación del dispositivo y el resultado es determinado por la configuración de permisos declarados anteriormente. La longitud puede ser null en los siguientes casos:

• La ubicación está desactivada en la configuración del dispositivo.

• El dispositivo nunca registró su ubicación.

• Se reiniciaron los Servicios de Google Play en el dispositivo y ningún cliente de proveedor de ubicación combinada activo solicitó la ubicación.

Cambiar la configuración de la ubicación


La información de ubicación puede permitir a los usuarios orientarse mientras caminan o conducen. Además de la ubicación geográfica (latitud y longitud), se puede mostrar otra información, como rumbo, altitud, velocidad, etc. Todos estos parámetros están disponibles en el objeto Location. Para esto propósito se puede hacer uso del método requestLocationUpdates().

Según el formato de la solicitud, se invoca al método de devolución de llamada LocationCallback.onLocationResult() y se le pasa una lista de objetos Location, o bien se le envía un PendingIntent que contiene la ubicación en los datos extendidos. Para obtener la actualización con el método de devolución de llamada LocationCallback. Se debe llamar a requestLocationUpdates() y pasarle la instancia del objeto LocationRequest y una LocationCallback. A continuación, se define un método startLocationUpdates().




Para definir la devolución de llamada de las actualizaciones de ubicación, se invoca el método de devolución de llamada LocationCallback.onLocationResult(). El argumento entrante tiene un objeto Location de lista que contiene la latitud y la longitud de la ubicación.



Para detener las actualizaciones de ubicación, debería considerarse si la actividad sigue enfocada o no, esto con el fin de reducir energía. Para ello se hace uso del método onPause(), llamando a removeLocationUpdates() y pasándole como parámetro un LocationCallback.



A continuación, haciendo uso de un valor booleano en el requestingLocationUpdates, se realiza un seguimiento para saber si las actualizaciones de ubicación están activadas en ese momento y se las activa de ser necesario.



Para guardar el estado de la actividad en caso de que la configuración del dispositivo cambie, se debe almacenar toda la información necesaria para recrear la actividad. Una forma de hacerlo es mediante un estado de instancia almacenado en un objeto Bundle y realizando la devolución de llamada onSaveInstanceState().



A continuación, se define un método updateValuesFromBundle() para restablecer los valores guardados de la instancia anterior de la actividad si están disponibles.



 

Acceder a ubicación en segundo plano


En caso de que la aplicación requiera ubicación en segundo plano (como al implementar geovallados), existe una lista de tareas para identificar la lógica de acceso:


• Usar los permisos ACCESS_COARSE_LOCATION y ACCESS_FINE_LOCATION. A continuación, verificar que la app requiera esos permisos de ubicación. En Android 10 (API 29) y posteriores, buscar también el permiso ACCESS_BACKGROUND_LOCATION. Llamarlos en una función.

• Definir el uso de la API de acceso a la ubicación dentro del código.

• Si la aplicación usa un SDK o una biblioteca que accede a la ubicación, dicho acceso se atribuye a la aplicación por lo que la mejor opción es revisar la documentación de la librería.



 

Geovallas


El geovallado combina el conocimiento de la ubicación actual del usuario con el conocimiento de la proximidad del usuario a ubicaciones de interés. La latitud, la longitud y el radio definen una geovalla, que crea un área circular, o valla, alrededor de la ubicación de interés. El máximo de geovallas activas es de 100 por app y por usuario de dispositivo.



Para crear objetos de geovallas el primer paso es usar Geofence.Builder a fin de crear una geovalla y configura el radio, la duración y los tipos de transición deseados.


Para especificar las geovallas a supervisar y sus activadores iniciales se hace uso de la clase GeofencingRequest y su clase anidada GeofencingRequestBuilder.

Para agregar geovallas, se usa el método GeofencingClient.addGeofences(), que proporciona el objeto GeofencingRequest y el PendingIntent.


Existe una lista de prácticas recomendadas para la implementación del geovallado:

• Reducir el consumo de energía.
• Elegir el radio óptimo para la geovalla.
• Explicar a los usuarios el porqué de las geovallas.
• Usar el tipo de transición de permanencia para reducir el spam de alertas.
• Volver a registrar geovallas solo cuando sea necesario.
• Solucionar problemas relacionados con el evento de entrada a la geovalla



 

Agregar mapas


La API de Google Maps para Android, permite incluir mapas e información personalizada de mapeos en una aplicación. Las principales funciones que permite esta API son las siguientes:


• Agregar mapas: 3D, de interiores, híbridos, de terreno, mosaicos basados en vectores para dibujos y almacenamiento en cache, transiciones animadas, etc.


• Personalizar mapas: Definir colores o íconos personalizados, incluir polilíneas y polígonos, proporcionar superposiciones de imágenes.


• Controlar la vista del usuario: Controlar las propiedades de rotación, inclinación, zoom y desplazamiento.


• Agregar Street View: Permite explorar el mundo a través de vistas panorámicas de 360°, controlar el zoom y la orientación, animar los movimientos de cámara.



 

Conclusiones


• Al momento de desarrollar una aplicación que requiera servicios de ubicación, se debe tener muy presente solicitar los respectivos permisos al usuario. Esto con la finalidad de ser transparentes con los objetivos de la aplicación.

• Librerías como Google Maps Android API, permiten implementar funcionalidades de georreferenciación en una aplicación móvil de manera rápida y simple.

• Agregar geovallados a una aplicación es un proceso sencillo debido a la cantidad de documentación disponible. Sin embargo, uno de los principales retos es el consumo de recursos que debe ser manejado apropiadamente con el fin de optimizar la aplicación.
